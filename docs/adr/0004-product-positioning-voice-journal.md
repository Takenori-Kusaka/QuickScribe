# ADR-0004: プロダクトポジショニング = 思考整理ボイスジャーナル

- Status: Accepted
- Date: 2026-06-15
- Deciders: Takenori Kusaka

## Context

競合調査の結果、「タスクバー常駐 × ローカル × Win/Linux × 音声→AI整形 × ホットキー」という
技術構成は OpenWhispr/Whispering/Hyprnote 等の OSS で既に実装済みで、技術的新規性は無い。
さらに Microsoft が Windows 2026 にオンデバイスAIディクテーションをOS統合予定で、基礎レイヤーは
コモディティ化する。一方で「思考整理・自己理解の用途特化」は Mem/Reflect のみ（クラウド依存・
Linux非対応）で、ローカル×非Apple×思考整理の交点は無人。

## Decision

QuickScribe を **「思考整理・自己理解のためのローカル完結ボイスジャーナル」** と定義する。
コア価値は文字起こし精度ではなく、ニュアンスを残しつつ思考を整理する「整形の知性」とする。

技術（文字起こし精度・OS対応・ローカル動作）を差別化の旗印にしない。
用途特化・ニュアンス保持・プライバシー約束・物理ボタン統合体験の4軸に投資する。

## Consequences

- ペルソナ/ストーリーブック/仕様/テスト観点は「ジャーナリング体験」を主軸に設計する。
- 「リッチすぎず簡便」のバランスが最重要のプロダクト設計課題となる（早期に縦切りで実証する）。
- 「QuickScribe」という名称が本質とズレる可能性があり、改名を別途検討する（[vision](../vision.md) オープン課題）。
- 競合と機能比較で殴り合わない。比較されたら「思考の相棒か、入力ツールか」の土俵の違いで応答する。
