# ADR-0022: whisper モデルカタログの精選（言語別ガイド）（#26 / #403）

- Status: Accepted
- Date: 2026-07-04
- Deciders: Takenori Kusaka
- 関連: [ADR-0021 ローカルファースト既定], [ADR-0006 スコープ完全性ポリシー], [ADR-0002 ローカルSTT], [product-positioning]
- 対象 Issue: #26（日本語精度ベンチ）/ #403（非機能ベンチ）

## コンテキスト

コア価値は「文字起こし精度」そのものではなく、ニュアンスを残す整形の知性（[product-positioning]）。
だが**性能が低すぎるモデルは後段の整形負担を増やすだけ**で、選択肢に並べる価値が薄い。
CLAUDE.md「リッチすぎると簡便でなくなる」＝**不要な選択肢を増やさない**という設計原則に照らし、
モデルカタログ（`src-tauri/src/model.rs`）を実測に基づき精選する。

判断材料として、本人音読のパブリックドメイン作品3点で日本語CER（文字誤り率）を実測した
（`src-tauri/tests/fixtures/ja-accuracy` / `scripts/cer_ja.py` / CI: `perf.yml` 日本語精度ジョブ）。

| サンプル | tiny | base | kotoba-q5 |
|---|---|---|---|
| 銀河鉄道の夜 | 45.1% | 38.0% | **26.1%** |
| 蜘蛛の糸 | 58.6% | **47.9%** | 57.1% |
| 吾輩は猫である | 67.1% | 46.1% | **31.6%** |
| **平均** | **56.9%** | **44.0%** | **38.3%** |

- **tiny は日本語で base に完全劣位**（全3点で悪化）。日本語では選ぶ理由がない。
- **kotoba-q5 は日本語平均で最良**だが、素材により**幻覚（反復・脱落）で base より悪化**しうる
  （蜘蛛の糸）。既定に据えつつ、頑健なフォールバックが要る。
- 注: 原文へのルビ混入で絶対CERは悲観側。相対/回帰指標として扱う（N=3）。
- 数値は CI 実測で確定済み（`docs/perf/baseline.md` が SSOT。回帰ゲート基準は `docs/perf/ja-cer-baseline.json`）。

## 決定

1. **カタログ順を用途優先で並べ替える**（既定 base を先頭に維持）:
   `base（既定・頑健）→ kotoba-q5（日本語推奨）→ kotoba → small → medium → tiny`。
2. **ラベルに言語別の適否を明示**する（実測に基づく）:
   - `tiny`: 「英語・下書き向き / 日本語は低精度で非推奨」
   - `small` / `medium`: 「多言語・英語向き」
   - `kotoba-q5`: 「日本語精度◎・日本語推奨」、`base`: 「頑健」。
3. **モデルの削除はしない**（[ADR-0006]）。tiny/small/medium も速度・英語・低リソース用途では有効。
   「削る」のではなく**用途別に正しい選択肢へ導く**ことで、簡便さと網羅性を両立する。

## 検討したが採らなかった案

- **tiny を日本語UIから除去/非表示**: 実測上は日本語で劣位だが、英語・下書き・低スペック機での
  最速選択肢として価値が残る。除去は選択肢の一方的縮小（[ADR-0006]）に当たるため採らない。
  UIラベルで非推奨を明示するに留める。
- **言語別にカタログを出し分け**: 実装コスト増。現状はラベルのガイドで十分機能する。将来 UI が
  言語別フィルタを持つなら再検討（後続の別判断）。

## 影響

- `src-tauri/src/model.rs`（順序・ラベル）。既定は base のまま、日本語既定 kotoba-q5 は [ADR-0021] 準拠。
- 既存ユーザーの保存済みモデル選択には非破壊（id 解決は不変。未知idは base フォールバック）。
- 精度回帰は `perf.yml` 日本語精度ジョブで継続監視（#26 / #403）。
