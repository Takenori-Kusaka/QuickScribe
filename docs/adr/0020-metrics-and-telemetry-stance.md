# ADR-0020: 採用計測はサーバー側公開統計のみ、アプリ内テレメトリは持たない（#60）

- Status: Accepted
- Date: 2026-07-02
- Deciders: Takenori Kusaka
- 関連: [ADR-0004 プロダクト位置づけ], [ADR-0019 プライバシー可視化], [product-positioning]
- 対象 Issue: #60 (S9.6 計測)

## 背景・課題
プロダクトの採用状況(ダウンロード数・流入)を把握したい。一方でコア価値の一つは
**ローカルプライバシー**であり、README/プライバシーポリシーで「解析・トラッキング・
テレメトリを行わない」と明言している。計測手段がこの約束と矛盾してはならない。

## 決定
1. **アプリ内テレメトリは実装しない**（利用状況の収集・送信を一切しない）。既存の約束を維持する。
2. 採用状況は **GitHub Releases のアセットDL数**（サーバー側の公開統計・個人を追跡しない）で測る。
   集計は `src/lib/release-metrics.ts`（単体テスト済み）＋CLI `scripts/release-metrics.mjs`
   (`npm run metrics`)。updater用 `latest.json`・署名・チェックサムはDL統計から除外する。
3. サイトの流入は、導入する場合も **クッキーレスでPIIを持たない集計**（例: Plausible相当の
   ページビュー/リファラのみ）に限定する。個人を識別する解析は採用しない（導入時は別途ADR）。
4. 「オプトインのアプリ内計測」は、上記のプライバシー約束と簡便さ(コア課題)の観点から**採らない**。

## 根拠（なぜ）
- 「テレメトリなし」は明文化した対ユーザー約束であり、破ると信頼と差別化(プライバシー)を毀損する。
- DL数はサーバー側で十分に採れ、ユーザーを一切追跡せずに採用の趨勢を把握できる。

## 反証可能性 / 未決
- DL数だけでは「継続利用/離脱」が見えない。将来それが意思決定に必須になった場合は、
  **明示同意・ローカル完結・匿名**を満たす最小計測の可否を新ADRで検討する（既定では持たない）。

## 影響
- 実装: `src/lib/release-metrics.ts` + `scripts/release-metrics.mjs` + `npm run metrics`。
- ドキュメント: README/プライバシーの「テレメトリなし」記述と整合。
