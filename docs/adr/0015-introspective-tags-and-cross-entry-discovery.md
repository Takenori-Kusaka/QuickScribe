# ADR-0015: 内省タグと横断発見（S4.3・段階実装）

- Status: Accepted（Phase 0=タグ付与 実装。Phase 1/2 は後続）
- Date: 2026-06-27
- Deciders: Takenori Kusaka
- 関連: [ADR-0006 スコープ規律], [S4.2 出力形式](../specs/s4-2-output-format/requirements.md)
- 対象 Issue: #34（Epic E4 #11）。製品ビジョン「思考整理・自己理解」の核。

## 背景・課題

S4.3 は「タグ付けと過去エントリ横断の気づき」。コア価値「自己理解」は、蓄積したエントリを**後から束ねて見返す**ことで生まれる。現状、保管庫は保存のみで一覧・横断の手段がない。
フル実装（タグ＋一覧＋横断発見）は大きいため、ADR-0006 に従い**削らず段階化**する。

## 決定

1. **タグはエントリのメタデータとして保存する**。Markdown では YAML フロントマター `tags: ["a","b"]`（**Obsidian 互換**＝外部ツールで即横断検索/グラフ化できる＝ローカル完結の強み）。プレーンテキストでは末尾に `Tags: a, b` 行を付す（形式に依らずタグが残る）。
2. **段階実装**:
   - **Phase 0（本ADRで実装）**: 整形結果にタグを付与して保存。タグ入力UI＋ md/txt 双方へのタグ書き込み。
   - **Phase 1（後続）**: 保管庫内エントリの一覧表示＋タグ/全文での絞り込み（アプリ内の横断導線）。
   - **Phase 2（後続）**: 横断発見（複数エントリから問い・傾向・気づきをAIが抽出）。コア価値の最終形。
3. **既定は非破壊・低摩擦**。タグは任意。未入力なら従来どおり。核心課題「リッチすぎず簡便」に配慮し、タグUIは目立たせすぎない。

## 結果・トレードオフ

- **Pro**: Phase 0 だけでも md=Obsidian 互換で外部横断が即可能。ビジョン直結。段階で着実。
- **Con**: Phase 0 単体ではアプリ内の一覧/発見導線は無い（外部ツール頼り）。→ Phase 1 で解消。
- **却下案**: ① 独自タグDB → mdフロントマター（標準・可搬）に劣る。② タグを必須化 → 低摩擦原則に反する。

## 「考えが変わる条件」
- ユーザーがアプリ内での一覧/検索を強く求める → Phase 1 を前倒し。
- タグの自由記述で表記揺れが課題化 → 既存タグのサジェスト（Phase 1付帯）を追加。
