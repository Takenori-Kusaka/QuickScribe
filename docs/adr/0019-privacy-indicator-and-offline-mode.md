# ADR-0019: プライバシー状態の可視化と「オフラインにする」導線（#465）

- Status: Accepted
- Date: 2026-07-01
- Deciders: Takenori Kusaka
- 関連: [ADR-0004 プロダクト位置づけ], [ADR-0005 BYO-Cloud], [product-positioning]
- 対象 Issue: #465 / 派生: #464（プライバシー文言の過剰主張是正）

## 背景・課題
差別化の一つは「ローカルプライバシー」。しかし既定の整形/STTはクラウド前提（BYO鍵 / ADR-0005）で、初期のUI文言は「話した言葉はこの端末から出ません」と**実態（クラウド選択時は送信される）に反する過剰主張**だった（#464で是正）。コア価値を誠実に提供するため、現在の構成が完全オンデバイスかクラウド送信を伴うかを可視化する。

## 決定（第1弾）
1. **プライバシー状態インジケータ**を設定先頭に常時表示する。`isFullyLocal = (整形=ローカルOllama) かつ (STT=ローカルwhisper)` のときのみ「オンデバイス完結」、それ以外は「クラウド送信あり」と表示する（誇張しない）。
2. クラウド時のみ **「オフラインにする」** ワンクリック導線（provider=ollama / stt=local へ即切替）を提供する。
3. **既定は変更しない**（クラウド既定のまま）。既定の変更・Ollama同梱・送信時の明示同意ダイアログは本ADRのスコープ外（後続で別途決定）。

## 根拠（なぜ）
- 「ローカル完結」はオプション（ローカルwhisper＋Ollama選択時）であり既定ではない、という事実をUIで正しく伝える（#464の是正の実体化）。
- モードを1トグルに留め、選択肢を増やさない（コア課題「リッチすぎると簡便でなくなる」）。

## 反証可能性 / 未決
- **既定をローカルにするか**（プライバシー優先）vs クラウド既定（手軽さ）は要判断。変更時は Deciders 承認＋本ADRの更新 or 新ADR。
- ローカルLLM（Ollama）の ja↔en 整形品質が実用水準かは別途検証（[output-language-feature] 連動）。

## 影響
- 実装: [#480](https://github.com/Takenori-Kusaka/QuickScribe/pull/480)。
- 後続: オフライン固定モード（クラウド選択不可化）・オンデバイス/クラウドの明示同意・Ollama同梱は #465 本体で継続。
