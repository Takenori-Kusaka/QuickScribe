# ADR-0010: v0.1.0 のゲート条件を「法的MUSTのみ」に限定（品質ブロッカーは v1.0.0 へ）

- Status: Accepted
- Date: 2026-06-21
- Deciders: Takenori Kusaka
- Amends: [ADR-0009](0009-release-versioning-and-1.0-scope.md)
- Relates to: [ADR-0006](0006-scope-completeness-policy.md), [ADR-0008](0008-licensing-and-distribution.md)

## Context

[ADR-0009](0009-release-versioning-and-1.0-scope.md) は v0.1.0 のゲートに品質ブロッカー7項目
（ライセンス整備・README刷新/用語/エラー文言・アクセシビリティ・サプライチェーン/リリースノート・
非機能通知/オンボーディング・性能ベースライン・ドキュメント整備）を**すべて**束ね、「完了まで v0.1.0 を
切らない」と定めていた。

しかしこれは Decider 本来の定義と矛盾する。ADR-0009 自身が「**顧客が離脱する理由を解消し、提供すべきもの・
公開OSSとしての社会的責任（ライセンス/帰属、アクセシビリティ、プライバシー、ドキュメント）をすべて満たして
初めて v1.0.0**」と述べている。すなわちアクセシビリティ・ドキュメント・オンボーディング・性能等の品質磨きは
**納品グレード＝v1.0.0 の条件**である。

v0.1.0 の本質は「**未実装機能が約25ストーリー残るため 1.0 にできない**」という*正直なバージョン表記*であり
（ADR-0009 Context）、重い品質バー（納品グレード）を課す対象ではない。ADR-0009 は「正直な低バージョン」と
「納品グレード」を1つのゲートに過剰に詰め込んでいた。本 ADR でこれを是正する。

## Decision

### v0.1.0 のゲート ＝ 「法的MUST」のみ
配布の*法的義務*に限定する（バージョンに依らず「何かを配布する時点」で必要なため）。具体的には ADR-0009 の
品質ブロッカー #1 に相当する次の3点のみ:

1. **ライセンス明記**: README/`Cargo.toml`/`package.json` を MIT に統一。README の「ライセンス:TBD」が
   LICENSE(MIT) と矛盾している不整合を解消する（=correctness）。
2. **第三者ライセンス NOTICE 同梱**: whisper.cpp(MIT) / libopus(BSD) / tauri 等、バイナリ配布に伴う
   帰属表記義務を `THIRD-PARTY-NOTICES` として `bundle.resources` に同梱（[ADR-0008](0008-licensing-and-distribution.md)）。
3. **`deny.toml`**: ライセンス allowlist を強制し、非互換ライセンスの混入を CI で防ぐ（ライセンス整備の一部）。

### 品質ブロッカー #2〜#7 は v1.0.0 へ再分類
次は「社会的責任／提供すべきもの」= Decider 定義の v1.0.0 条件。**v1.0.0 マイルストーン**に紐づけ、未実装
25ストーリーと並走して達成する（v0.1.0 のゲートにはしない）:

- README のエンドユーザー向け刷新、用語統一（「成形」→「整形」）、ユーザー向けエラー文言化。
- アクセシビリティ P0/P1（WCAG 2.0 / JIS X 8341-3:2016 AA）。
- サプライチェーン衛生（`Cargo.lock` コミット＋ CI の `|| true` 除去ゲート化）＋リリースノート自動化。
- 非機能のユーザー通知（対応形式・入力/録音長上限・初回DL告知）＋初回オンボーディング。
- 性能ベースライン測定。
- ドキュメント整備（アーキ概要・未記録決定の ADR 化・ADR-0005 Status 更新・as-shipped 明記）。

> 注: サプライチェーンのうち **`deny.toml`** はライセンス強制を兼ねるため v0.1.0（法的MUST）側に置く。
> 一方 `Cargo.lock` コミットや `npm audit`/`cargo audit` の `|| true` 除去は*セキュリティ衛生*であり
> 法的MUSTではないため v1.0.0 とする。

## Consequences

- **v0.1.0 を切る条件は「法的MUST（上記3点）」の完了のみ**。ADR-0009 が課していた重いバーを大幅に軽量化。
- 品質ブロッカー #2〜#7 は v1.0.0 マイルストーンで追跡（未実装機能と同じ扱い）。
- [ADR-0006](0006-scope-completeness-policy.md) のスコープ不変と整合（機能の削除ではなく*順序付けの精緻化*）。
- `v0.0.x-alpha.N` は引き続き開発中の自動更新検証用プレリリース。法的MUST完了後、最初の正式節目を v0.1.0 とする。
- 本決定は ADR-0009 の「v0.1.0 に含める品質ブロッカー」節を **amend** する（ADR-0009 はその旨ポインタを追記）。
