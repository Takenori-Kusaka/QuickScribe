# ADR-0023: 習慣ナッジ＝起動アンカーの opt-in ローカル通知（S9.4 #58）

- Status: Accepted
- Date: 2026-07-04
- Deciders: Takenori Kusaka
- 関連: [ADR-0004 プロダクト位置づけ], [ADR-0019 プライバシー], [ADR-0021 ローカルファースト既定], [product-positioning]
- 対象 Issue: #58（S9.4 ローカル通知・習慣ナッジ）

## コンテキスト

思考整理の習慣化を助けたい（リテンション）。一方でコア価値は「簡便さ」と「ローカルプライバシー」で、
CLAUDE.md「リッチすぎると簡便でなくなる」が最大の制約。通知は**やり方を誤ると即ノイズ／監視感**になり、
プロダクトの信頼を損なう。サーバーは持たない方針（[ADR-0020]）。

技術制約: デスクトップアプリはバックグラウンド常駐スケジューラを持たない（常駐トレイはあるが、OSレベルの
定時ジョブは持たない＝サーバーレス方針とも整合）。任意時刻のリマインドは実装コスト・監視感の両面で不利。

## 決定

1. **アンカー＝アプリ起動**にする。自動起動（[tauri-plugin-autostart] 既存）と併用すれば「ログイン＝毎日の起点」
   が自然なアンカーになる。定時スケジューラは持たない。
2. **既定 OFF（opt-in）**。設定「習慣リマインダー」を明示的に ON にした人だけ通知する。ON にした瞬間にのみ
   OS 通知権限を要求し、拒否されたら自動で OFF に戻す（勝手に鳴らさない）。
3. **forgiving 発火条件**（純粋関数 `shouldNudge`）:「継続中のストリークがあり、今日まだ記録していない」ときだけ。
   - 新規/離脱ユーザー（ストリーク0）は促さない＝罪悪感を煽らない。守る価値のある習慣だけ保全する。
   - 1日1回まで（`lastNudgeDate` で重複防止）。起動のたびには鳴らさない。
4. **完全ローカル**。ネットワーク送信なし。通知本文も端末内 i18n（ja/en/zh/es）。

## トレードオフと緩和（[ADR-0006]）

- **起動アンカーの弱点**: アプリを起動しない日は通知が出ない。→ 自動起動と併用すれば毎日ログイン時に発火。
  「サーバーレス・非監視・低ノイズ」を優先した意図的な割り切り（定時プッシュは将来別判断）。
- **通知疲れリスク**: opt-in＋1日1回＋ストリーク保全時のみ、で最小化。トーンも「急かさない」文言に統一。

## 実装

- 判定: `src/lib/streak.ts` `shouldNudge`（純粋・TDD）。副作用: `src/lib/nudge.ts` `maybeNudge`（依存注入でテスト）。
- 発火: `App.svelte` onMount で `list_entries` の記録日を渡して `maybeNudge`。
- 設定: `nudgeEnabled`（既定 false）を settings-persist に追加。UI は一般タブ。
- 基盤: `tauri-plugin-notification`（Rust/JS）＋ capability `notification:*`。
