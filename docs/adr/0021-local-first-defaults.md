# ADR-0021: ローカルファースト既定（整形=Ollama / 日本語STT=kotoba-whisper）（#465 / #511）

- Status: Accepted
- Date: 2026-07-04
- Deciders: Takenori Kusaka
- 関連: [ADR-0004 プロダクト位置づけ], [ADR-0005 BYO-Cloud], [ADR-0019 プライバシー状態の可視化とオフライン導線], [product-positioning]
- 対象 Issue: #465（ローカル完結モードの実体化）/ #511（STTモデルUX）

## コンテキスト

コア価値の差別化は「ローカルプライバシー」（[ADR-0004] / product-positioning）。
[ADR-0019] は当面「既定は変更しない（クラウド既定のまま）」とし、既定変更は
「Deciders 承認＋本ADRの更新 or 新ADR」を条件としていた（ADR-0019 §結論-3, §22）。

本ADRで Decider（Takenori Kusaka）がその条件を満たし、**既定をローカルファーストへ変更**する
判断を下す。誠実にコア価値（機密性の高い思考を端末外に出さない確信）を一級市民として提供するため。

## 決定

1. **整形プロバイダの既定を `ollama`（ローカル）へ変更**する（従来 `gemini`）。
   新規ユーザーは既定でオンデバイス整形になり、鍵不要でプライバシーが守られる。
2. **STT の既定はローカル whisper のまま**（変更なし）。日本語UIユーザーの whisper 既定モデルを
   **`kotoba-q5`（日本語特化・約538MB）** とする（従来一律 `base`）。非日本語UIは `base` のまま。
3. 上記は [ADR-0019] §結論-3「既定は変更しない」を**本ADRで置き換える**（supersede）。

## トレードオフと緩和（ADR-0006: リスクは削らず設計・段階で対処）

- **整形=ローカル既定は Ollama 稼働が前提**。未導入だと初回整形が失敗する。
  緩和: 整形失敗は明確な i18n エラー（`E_REFINE_*` / #462）で通知済み。オンボーディング・設定に
  「ローカル整形には Ollama が必要」の導線を添える（本ADRのスコープ。Ollama 同梱/自動セットアップは
  引き続き #465 の「(検討)」項目＝後続の別判断）。ユーザーはクラウドプロバイダへ即切替も可能。
- **日本語 kotoba 既定は初回DLが大きい（約538MB / base は約148MB）**。
  緩和: モデルは非同梱・初回自動DL（`tip_model_download` で明示済み）。サイズは UI ラベルに表示。
  「ワンボタン→着地の摩擦ゼロ」（vision）より**日本語精度**を優先する Decider 判断。base への変更も1操作。

## 段階実装

- Phase A（本ADRと同時 / #511）: 日本語UIの whisper 既定を `kotoba-q5` に。サイズ/非同梱コピーは既存。
- Phase B（#465）: 整形既定を `ollama` に変更＋オンボーディング導線。
- Phase C（後続・別判断）: Ollama 同梱/自動セットアップ、クラウド送信時の明示同意ダイアログ。

## 影響

- `src/lib/settings-persist.ts` の新規ユーザー既定（`provider` / `whisperModel`）。
- 既存ユーザーは保存値が優先されるため影響を受けない（非破壊）。
