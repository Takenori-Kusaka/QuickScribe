# ADR-0009: リリースのバージョニングと v1.0.0 スコープ

- Status: Accepted
- Date: 2026-06-20
- Deciders: Takenori Kusaka
- Relates to: [ADR-0006](0006-scope-completeness-policy.md), [ADR-0008](0008-licensing-and-distribution.md), [4 backlog](../planning/4-backlog.md), [3.6 release-ops](../planning/3.6-release-ops.md)

## Context

最終自己レビューの結果、Epic/Story バックログ（[4 backlog](../planning/4-backlog.md)）のうち
**17 ストーリーは実装・出荷済み**だが、**約25 ストーリーが未実装**であることが確認された
（システム音声ループバック録音、入力デバイス選択、鍵のOSセキュアストレージ、整形スタイル切替/
ローカルLLM、出力形式・メタデータ・内省タグ、PC自動起動、クラウドSTT/日本語精度ベンチ、
ドキュメントサイト/オンボーディング、コード署名/SECURITY.md 等）。

[ADR-0006](0006-scope-completeness-policy.md)（スコープ完全性ポリシー）により、これら未実装機能は
**「最終ゴールの機能集合」に含まれ、削除・縮小してはならない**。したがって現状を「全機能完成」を
意味する **v1.0.0 とすることはできない**。一方で、現行の出荷機能は実機で動作し、品質を納品グレードへ
引き上げる価値がある。バージョン番号の意味づけと、いま切るリリースの位置づけを確定する必要がある。

## Decision

### バージョニングの意味づけ（SemVer 0.y を採用）
- **v1.0.0 = 計画スコープ（[4 backlog](../planning/4-backlog.md) の全 Epic/Story）の完成 ＋ 納品グレード品質**。
  顧客が離脱する理由を解消し、アプリとして提供すべきもの・公開OSSとしての社会的責任
  （ライセンス/帰属表記、アクセシビリティ、プライバシー、ドキュメント）を**すべて満たして初めて 1.0.0**。
- **現状で切るリリースは v0.1.0**（0.y 系のマイナー）。「現行の出荷機能 ＋ 最終自己レビューで洗い出した
  品質ブロッカーを解消した、まともな公開リリース」と位置づける。未実装機能が残るため 1.0.0 ではない。
- これまでの `v0.0.x-alpha.N` は開発中の自動更新検証用プレリリース。**v0.1.0 を最初の「正式な節目」リリース**とする。

### v0.1.0 に含める品質ブロッカー（最終自己レビュー由来。完了まで v0.1.0 を切らない）
1. ライセンス整備：README/Cargo.toml/package.json を MIT に統一、第三者ライセンス NOTICE 同梱、`deny.toml` でライセンス強制。
2. README をエンドユーザー向けに刷新、用語統一（「成形」→「整形」）、ユーザー向けエラー文言化（生APIレスポンス露出の解消）。
3. アクセシビリティ P0/P1（WCAG 2.0 / JIS X 8341-3:2016 AA 主要項目：aria-live 状態通知、設定モーダルの dialog 意味論＋フォーカス管理、コントラスト、フォーカス可視、forced-colors/reduced-motion）。
4. サプライチェーン：Cargo.lock コミット＋ CI ゲート化（`|| true` 除去、`deny.toml`）。リリースノート自動化（generate-notes）＋過去版手当て。
5. 非機能のユーザー通知：対応音声形式・入力サイズ/録音長の上限の実装と提示、初回モデルDL（約150MB）の事前告知、初回オンボーディング。
6. 性能ベースライン測定（whisper 文字起こしの基準値を記録）。
7. ドキュメント整備：アーキテクチャ概要、未記録の重要決定の ADR 化、ADR-0005 の Status 更新、as-shipped 状態の明記。

### 未実装の計画機能の扱い
- **削除・クローズしない。** GitHub の **`v1.0.0` マイルストーン**に紐づけ、open のまま 1.0 へ向けて追跡する。
- 実装・出荷済みの 17 ストーリー（102 issue）は**証跡を確認のうえクローズ**し、`v0.1.0` マイルストーンに整理する。

## Consequences

- いま切るタグは **v0.1.0**。`release.yml` の `v*` トリガはそのまま機能する。
- 「全 issue クローズ」は 1.0.0 の条件であり、v0.1.0 の条件ではない（未実装機能の issue は open のまま）。
- 品質ブロッカーは GitHub Flow に従い専用ブランチ＋PR＋CI で順次解消し、すべて完了後に v0.1.0 を公開する。
- 本決定は [ADR-0006](0006-scope-completeness-policy.md) のスコープ不変原則と整合する（縮小ではなく**順序付け**）。
