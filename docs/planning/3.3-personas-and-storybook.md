# 3.3 ペルソナ / ストーリーブック（DDDドメイン策定）

## ユビキタス言語辞書（初版）

| 用語 | 定義 | 備考 |
|---|---|---|
| ジャーナルエントリ(Entry) | 1回の録音から生成される思考の記録（音声→文字起こし→整形の単位） | 集約ルート候補 |
| キャプチャ(Capture) | 録音し文字起こしする行為 | TriggerInput起点 |
| 整形(Refine) | 逐語⇄要約⇄ブレストの知的変換 | コア価値。FormattingEngine |
| 逐語(Verbatim) | ニュアンスを残した生の文字起こし | 整形スタイルの一つ |
| 内省タグ(Reflection tag) | エントリに付く主題/感情の手がかり | 後の横断発見に使う |
| 保管庫(Vault) | エントリ群が着地するローカルフォルダ | 既定: ドキュメント/QuickScribe |
| 整形エンジン/文字起こしエンジン | 差し替え可能な戦略 | [ADR-0005](../adr/0005-tech-stack.md)の4境界 |

> この語彙を requirements / テスト名 / コードに**貫通**させる（[3.4](3.4-spec-and-tdd-plan.md)）。

## 境界づけられたコンテキスト（DDD・初版）

- **Capture**（録音・デバイス・トリガー）
- **Transcription**（音声→テキスト、エンジン差し替え）
- **Refinement**（整形の知性＝コアドメイン）
- **Journal/Storage**（エントリ永続化・保管庫・検索）
- **Settings/Secrets**（フォルダ・ホットキー・鍵管理）

→ **コアドメイン = Refinement**（ここに設計努力を集中。他は支援/汎用）。

## ペルソナ（初版）

1. **内省したい開発者「ケンジ」**: Win/Linux併用、プライバシー重視、ホットキー/Stream Deck愛好。就寝前に一日を声で振り返り、考えを整理したい。クラウド送信に強い抵抗。
2. **書く前に詰まるライター「アヤ」**: アイデアは話すと出るが、キーボードだと止まる。話した断片を構造化された下書きにしたい。
3. **思考を整理したい経営者「タカ」**: 移動中に意思決定の迷いを声に出し、要点とトレードオフに整理したい。機密のためローカル必須。

各ペルソナの Job Story は [3.1コア価値](3.1-lean-canvas-and-planning.md) の型に従う。

## ストーリーブック（UATの素体・各視点を内包）

各シナリオは「状況→操作→期待」で、E2E/受入テストに変換可能にする（[3.5](3.5-waterfall-and-quality.md)）。

- **S1 就寝前の振り返り（顧客視点）**: ケンジがホットキーで録音→「今日の迷い」を話す→停止→**逐語＋要約**が保管庫に着地→読み返して安堵。
- **S2 摩擦ゼロの起動（顧客/UX視点）**: トレイ常駐から1アクションで録音開始。白紙パラリシスなし（既定エントリ）。
- **S3 プライバシーの確証（投資家/品質視点）**: タカが「これは本当に外に出ないか」を確認→ローカル完結とOSS監査可能性で安心（メッセージング[3.7](3.7-marketing.md)）。
- **S4 ブレスト整形（営業/差別化視点）**: アヤが断片を話す→整形で構造化下書き→「声で書くObsidian」の価値を体感。
- **S5 物理ボタン連携（差別化視点）**: Stream Deckボタンで開始/停止→整形済みメモ着地までの摩擦ゼロ。
- **S6 エンジン切替（開発/拡張視点）**: ローカルwhisper⇄クラウドSTT、BYO-LLM⇄ローカルLLMを設定で切替（[ADR-0002](../adr/0002-stt-engine-strategy.md)/[0005](../adr/0005-tech-stack.md)）。
