## 0. 問い設計(ADR-0007)

- **ジョブ接続(JTBD)**: 「When アプリ一覧/タスクバーで自分のジャーナルアプリを探すとき, I want to ひと目で『これは録音ツールでなく思考整理の相棒だ』と分かりたい, so I can 思考を吐き出す行為に迷わず入れる」。
- **意思決定駆動(VOI)**: この調査は「次に生成する画像プロンプトの方向(メタファー/色/構成)」という具体的意思決定を変える。どの案でも同じ絵を作るなら VOI=0 だが、競合サーベイの結果(マイク+波形は埋没・positioning と矛盾)が方向を反転させた → VOI > 0。
- **Heilmeier "Who cares?"**: アイコンは最初の接触点。「録音ユーティリティ」と読まれると ADR-0004 の差別化(用途特化×統合体験)が初手で崩れる。
- **反証可能性**: 各結論に「何が分かれば考えが変わるか」を併記(下記各節)。
- **原典主義**: Apple HIG / Material / MS Fluent / WCAG / Google 画像生成docs を一次で固定(sources/ 参照)。色意味論は最強一次(Jonauskaite 2020)で紫の不安定性を確認。

## 1. 設計原則チェックリスト(一次情報出典つき)

### 可読性・シルエット(3社合意)
- [ ] 単一コンセプト/単一フォーカル要素(MS は ≤2 metaphor、1つが理想)。出典: Apple/Material/MS。
- [ ] 最小形状・最小コーナー。出典: Apple "minimal number of shapes" / MS "as few shapes... as few corners"。
- [ ] 16-32px で可読(小サイズを明示的制約に)。出典: Apple "hard to discern at smaller sizes" / MS "legible at small sizes" / M3 "legible even at smaller scales"。
- [ ] 極細線・鋭角・微細ディテールを避ける。出典: Apple "avoid extremely thin line weights"。
- [ ] ラベル無しで何かを伝えるシルエット。出典: MS "tell what it represents without a label"。
- [ ] テキスト/文字を入れない。出典: Apple / MS "Letters and words... should be avoided"。
- [ ] 写実/UI複製を避ける。出典: Apple/Material/MS。
- [ ] セーフエリア/グリッド(48グリッド/ライブエリア、中央配置)。出典: MS 48×48 / Material live area / Apple centered。

### 色・コントラスト
- [ ] グリフ対地で隣接色に 3:1 以上、明/暗テーマ双方で「半分以上が 3:1」。出典: WCAG 1.4.11 / G207 / MS app-icon-design。
- [ ] **自己完結した塗りの角丸スクエア本体** + 内側反転グリフ(透過上の細グリフ浮遊を避ける)。根拠: 明/暗/壁紙のどれでも図地コントラストが固定される(WCAG 1.4.11 の運用)。
- [ ] グラデ・色処理を抑制しスケール/テーマに耐える。出典: MS "treatments to colors should be minimized"。

### フォースドカラー/HC
- [ ] グラデ・影に identity を依存させない(HC でグラデ/影は破棄)。出典: MDN forced-colors / MS "Avoid gradients in high contrast icons"。
- [ ] 2トーンのシルエットで成立すること。

### 透過親和性(本リポ scripts/process-brand-icon.py)
- [ ] 外側グロウ/ブルーム/ビネット/ドロップ影なし(中彩度ハローがゲート失敗の主因)。
- [ ] 本体は深く彩度の高い色(min(R,G,B)<170)。白は本体色に使わず内部グリフに限定。
- [ ] クリスプな硬い縁。たっぷり余白・中央配置。純白フラット背景で生成。
- [ ] 不透明割合 0.55-0.90 / 角は透過(alpha<=8)に収まる単一連結の本体。

反証条件: いずれかのプラットフォーム公式が上記を撤回/緩和したら(例: Apple Liquid Glass がフラット source を不要化)該当項目を見直す。

## 2. 色の方向(根拠と反証)

- 紫は意味が最も異文化的に不安定な色(Jonauskaite 2020, n=4,598)。「紫=内省」は最弱の論拠。加えて 2023-2025 の AI ブランドで紫グラデが過飽和 → ローカル/反クラウドAI の positioning と逆行。
- **推奨**: 紫系を捨てず規律する。マゼンタ紫(290-320°)→ **deep indigo/blue-violet(約250-275°)**。深い indigo タイル + 内側に白/淡グリフ。フラット〜準フラット。任意で控えめ teal アクセント。
- 反証条件: (a) 音声ジャーナル/ローカルAIニッチで indigo が既に密集なら差別化にならず温かい内省レーン(deep teal-green / warm clay)へ / (b) ユーザーテストで deep indigo が「冷たい企業AI」と読まれたら転換。

## 3. メタファー比較と推奨(詳細は §sources、評価は本文末の表)

- 現行(マイク+波形)は最大クリシェに直撃し positioning と矛盾 → 不採用方向。
- ジャーナリング/思考系競合はマイク・波形を使わず、成長/本/抽象で「思考の仕事」を表現(Rosebud/Day One が最強参照)。
- **推奨1案: もつれた糸→一本の滑らかな線**(変換=「整形の知性」を直接描く、競合空白)。次点: **芽吹き**(自己理解・市場実証, Rosebud)。
- 反証条件: 推奨1案が 16-24px で「ただの渦」に潰れる/Granola と近接しすぎると目視確認されたら、芽吹き or リップルへ。

## 4. 評価マトリクス(差別化/直感性/小サイズ耐性/実装容易性, 5段階)

| 案 | 差別化 | 直感性 | 小サイズ耐性 | 実装容易性 | 総評 |
|---|---|---|---|---|---|
| 現行 マイク+波形 | 1 | 4 | 3 | 4 | クリシェ直撃・positioning 矛盾。不採用方向 |

## 5. 画像生成プロンプトと次アクション
- 推奨1案(もつれ→直線)を主軸に、次点 5(芽吹き)も保険でプロンプト化(本ノートの呼び出し元 deliverable / 別途 prompt 集参照)。全プロンプトは scripts/process-brand-icon.py 互換(透過/フラット/グロウ無し/単一フォーカル/余白/deep indigo 本体)。
- 次アクション: プロンプトで各案 N 枚生成 → `python scripts/process-brand-icon.py` で透過化(品質ゲート通過を確認)→ 16/24/32/48px へ縮小して明暗タスクバー双方で目視 → 推奨1案がもつれの可読性で落ちる場合は次点へ。最終選定後 `npm run icons`。

## 6. 統合分析(2026-06-20 追補) — 5段階の掘り下げ

> 経緯: 初回12案＋単一マーク9案がユーザーに「チープ／とりあえず作ってみた」と却下。深掘り2調査を実施
> (`sources/logo-prompting-techniques.md` / `sources/competitor-logos-survey.md`、いずれも一次情報・出典URL付き)。
> 却下の真因が「フォーマット(SVG/raster)」でも「タイルの有無」でもないと判明したため、本節で再設計する。

### 6.0 却下の真因(確定)

1. **プロンプトが浅い**: キーワード列挙(`flat 2D vector app icon, indigo tile, white mark...`)は Google 公式 Gemini docs が「劣る／hyper-specific な叙述段落にせよ」と名指しで非推奨にしている悪手だった。
2. **明るい飽和ベタ地＋太字グリフ**が安っぽさの本体。競合の優れたタイルは**暗地＋淡/発光グリフ**(Obsidian `#6c31e3`/Apple Journal)。「タイルが安い」のではない。
3. **craft指定ゼロ**: squircle(超楕円・曲率連続)、monoline、ネガティブスペース、光学調整が未指定。
4. **参照画像conditioning未使用**。
5. プラットフォーム作法の混同: Windows タスクバーは自由形状透過も多いが、競合は App Store では自己完結タイルが主流。**「タイル/透過」より「マークが自前でコントラストを持つか」が本質**(WCAG 1.4.11)。

### 6.1 [レベル1] ポジショニングの錨(vision.md)

思考整理・自己理解のローカル完結ボイスジャーナル。コア価値=**「ニュアンスを残しつつ思考を整理する整形の知性」**。
ロゴが初手で伝えるべきは「録音ツール」ではなく「**思考整理の相棒**」。※名称 "QuickScribe" は維持で決着済みだが、**特定文字(Q)に意匠を縛らない**方が将来の柔軟性を保てて安全。

### 6.2 [レベル2] ユーザーに感じてほしい情緒(competitor survey の情緒列から逆算)

- **狙う**: 静謐 / 内省 / 知的で信頼できる道具性 / ローカルの落ち着き / ニュアンスの温度を「1点だけ」。
- **避ける**: クラウドAIの冷たさ・紫ネオン発光球(Reflect=positioning と真逆) / wellness の甘さ(Rosebud のレーン) / quirky な遊び心過多(Granola) / 説明的な literal さ(マイク・吹き出し)。
- 競合マップ上の空き: **deep indigo × 内省 × 道具性**。最直接 Rosebud は暖マゼンタ×ケアなのでレーンが割れ、差別化上も合理的。

### 6.3 [レベル3] メタファー(競合は全社「1段抽象化」している)

- 第一候補: **もつれ → 一本の滑らかな線**(コア価値「整形の知性」を直接体現・競合空白)。ただし却下教訓を踏まえ**「before/after の図解」にせず、ごく短い1つのほどけ目を持つ単一の連続 monoline** に抽象化(情報を盛らない)。
- 次点: **一筆のインク/筆致**(捕えた声が整った一文へ) / **静かな単一の弧**(声の一吐き)。
- 不採用: マイク/波形そのまま(literal・クリシェ)、Qモノグラム(改名リスク＋単純文字は今回チープに直結)。

### 6.4 [レベル4] 形と craft の設計原則(両調査の合流点)

- **暗 deep-indigo の squircle 地 ＋ 淡く発光する monoline グリフ**(Obsidian/Apple Journal の「ジャーナル文法」を参照)。明るいベタ地＋白ボールドは禁止。
- **単一・大胆・抽象シルエット**で 16px 耐性を確保(Day One/Bear/Notion が実証、細部過多の Otter/Reflect は一段劣る)。
- **monoline / 単一連続線 / ネガティブスペース**で描く(太字ベタ塗りグリフを捨てる)。
- **squircle**: cornerSmoothing≈0.6 / 角丸半径≈幅22.4% で曲率連続(G2)。普通の角丸タイルの安さを消す。
- **単色相規律**(deep indigo 250-275°を一貫)。グラデは使うなら**単色相内 or 暗地に発光**のみ(色相跨ぎの派手グラデは小サイズ・HC で破綻)。
- **光学調整**(円形要素は方形より一回り大きく、飛び出す角・線幅を縮める=Helena Zhang)。
- マークが自前でコントラストを持てば、最終形は**暗タイル**でも**透過自由形状**でも可(タスクバー目視で決定)。

### 6.5 [レベル5] 実行プロンプト仕様(Gemini現実解 — 鍵は Gemini のみ)

1. **叙述段落**で書く(キーワード列挙を廃止)。情緒・素材・光・構図・mood を文章で。
2. **参照画像conditioning**: Apple Journal/Obsidian 風「暗地＋発光グリフ」の参照を渡し、公式テンプレ "Transform ... into the artistic style of [reference]. Preserve original composition." 正方形参照を最後に置く(最後の画像のアスペクト比が採用される)。
3. monoline / negative space を明示、太字ベタ塗りを否定。
4. 生成後に **squircle 化＋光学調整**を後処理(`scripts/`)。`process-brand-icon.py` は暗地タイル対応に要調整(彩度マスク前提のため、暗 indigo 地＋淡グリフ用に閾値を見直す)。
5. 一発生成にせず**参照固定→1-2語だけ変えて反復**。16/24/32/48px 明暗タスクバー目視＋品質ゲート。
6. ネイティブ SVG が要る局面は Recraft が現状唯一の主力(将来オプション。今回は Gemini ラスター＋後処理で底上げ)。

### 6.6 推奨の結論

**暗 deep-indigo squircle ＋ 淡発光の単一 monoline マーク(ほどけ目1つを持つ連続線=整形の知性)**、単色相・内省/道具性の情緒。
これにより却下の真因(明地＋太字＋浅プロンプト＋craft欠如)をすべて潰し、競合空白(indigo×内省)に置く。
