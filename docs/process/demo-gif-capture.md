# デモGIF キャプチャ手順（README/サイト用）

> Status: Reference（2026-06-28）。READMEの「録音→文字起こし→整形」操作デモGIF（#55 残タスク）を実機で撮るための手順。
> 撮影後、ファイルを `docs/assets/demo.gif` に置けば README のプレースホルダ `<!-- DEMO ... -->` に差し込み、自動化側で最適化・埋め込みを行う。

## 0. 役割分担
- **あなた（実機）**: 実際にアプリを操作して画面録画 → GIF/MP4 を書き出す。
- **私（自動化）**: 受け取った素材の最適化（サイズ/フレーム削減）・READMEとサイトへの埋め込み・altテキスト整備。

## 1. 推奨ツール（Windows・無料）
- **ScreenToGif**（OSS・推奨）: `winget install --id NickeManarin.ScreenToGif -e`
  - 録画→フレーム編集→GIF/MP4書き出しが1アプリで完結。トリミング・速度調整・不要フレーム削除が容易。
- 代替: Xbox Game Bar（`Win+G`、MP4録画）→ 別途GIF変換。

## 2. 撮影の構図・段取り（15〜25秒目安）
ストーリーは **「話す → 文字になる → 整う」** の3拍。コア価値（整形の知性）が伝わる流れにする。

1. ウィンドウを既定サイズに。背景に個人情報が映らないよう注意。
2. 録音開始（グローバルホットキー or ボタン）→ 短く一言話す（例: 「最近やることが多くて、何から手をつければいいか迷っている。まず優先順位を整理したい。」）。
3. 停止 → 文字起こし結果が出る。
4. 「整形」実行 → ニュアンスを保ったまま整った文章になる様子を見せる。
5. （任意）用語補正の「✓ 用語チェック」を一瞬見せる。

撮影のコツ:
- マウス移動は最小限・ゆっくり。クリック箇所が分かるように。
- 待ち時間（文字起こし/整形の処理）は編集で**カット or 早送り**して冗長さを排除。
- 文字が読める速度に。最後の整形結果は1〜2秒静止で締める。

## 3. 書き出し設定（GitHub/READMEで軽快に表示）
- 解像度: 幅 **960〜1200px** 程度（等倍ウィンドウを縮小）。
- フレームレート: **10〜15fps**（GIFは高fpsだと肥大）。
- **目標ファイルサイズ: 5MB以下**（理想 2〜3MB）。GitHubのMarkdownで自動再生される。
- 色数を抑える/不要フレーム削除でサイズ圧縮。長尺になるなら MP4 も併用検討（GitHubはmp4も`<video>`で再生可）。

## 4. 提出方法
- 書き出した `demo.gif`（または `demo.mp4`）を **`docs/assets/`** に置く（無ければ作成）。
- 私に「置いた」と伝えてもらえれば、最適化のうえ README プレースホルダとサイト（index/guide）に埋め込み、#55 をクローズする。

## 5. プライバシー確認（撮影前チェック）
- [ ] 画面に実APIキー・個人ファイル名・メールアドレス等が映っていない。
- [ ] 保管庫の他エントリ一覧に私的内容が映っていない（デモ用に空フォルダ/ダミーで撮るのが安全）。
- [ ] 録音内容に個人情報を含めない。
