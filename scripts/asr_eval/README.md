# 日本語 ASR / 整形 評価スタック（ADR-0024）

QuickScribe の日本語文字起こし精度（CER）と、整形の「ニュアンス保持」を**公開コーパス＋統計的に妥当な指標**で評価するためのスクリプト群。設計の根拠と段階計画は [ADR-0024](../../docs/adr/0024-evaluation-redesign-cer-and-nuance.md) を参照。

> **アプリ本体（Tauri/Rust）はこのスタックに依存しない。** 評価は CI / ローカルの補助ツールで、すべてオフライン・ローカル完結で回す。

## 現状（第一スライス）＝評価コア

| ファイル | 役割 |
|---|---|
| `normalize_ja.py` | 日本語 CER 用テキスト正規化（括弧内除去→(任意)neologdn→NFKC→小文字化→約物/記号除去→**空白完全除去**）。参照/仮説へ同一適用。数字/かな吸収は既定OFF（実差を隠さない）。 |
| `cer.py` | 文字単位 CER（Levenshtein/参照長）、マイクロ平均、**発話単位ブートストラップ95%CI**（seed固定で決定的）。 |
| `test_asr_eval.py` | 上記の単体テスト（依存を増やさず plain assert）。 |

### テスト実行

```sh
cd scripts/asr_eval
python test_asr_eval.py
```

`neologdn` は任意。未導入なら正規化の該当段を自動スキップ（テストは `use_neologdn=False` で決定的に検証）。CI では `requirements.txt` で導入する。

## 段階計画（ADR-0024）

- **第一スライス（実装済み）**: 評価コア（正規化・CER・ブートストラップCI）。← 本PR
- **第二スライス**: Common Voice ja（CC0ミラー）を CI 実行時に取得（音声は**非コミット**）→ whisper.cpp で文字起こし → 本コアで CER＋95%CI を算出し `perf` レポートへ。既存の自前録音 fixture は smoke として残す。JSUT basic5000（DLのみ遵守）を回帰ベースラインに追加。フィラー保持率を独立計測。
- **第三スライス**: 意味保持（BERTScore-ja＋日本語NLI）と、黄金評価セットによるペアA/B（整形版 vs 要約版）。**指標の日本語妥当性は少数の人手相関で自前検証**（#577）。

## 方針の要点（なぜこう測るか）

- **CER 前に空白を完全除去**するのは日本語 ASR 評価の必須点（語間空白が無いため）。
- **数字表記・かな種別の吸収は既定OFF**。吸収すると CER は下がるがモデルの実出力差を隠す＝コア価値「実差を消さない」と不整合。
- **信頼区間が重なる差は「差なし」**として扱う（N=3 の脆弱な相対比較からの脱却）。
