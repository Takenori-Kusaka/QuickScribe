# 使い方

## 1. インストール

[ダウンロードページ](/download) から、お使いのOSのインストーラを入手して実行してください（Windows x64 / ARM64、Linux AppImage / deb）。
初回起動時、ローカル文字起こし用のモデルは初回録音時に自動ダウンロードされます。

## 2. 録音する

- ウィンドウの **「録音開始」** ボタン、または **グローバルホットキー**（既定 `Ctrl/Cmd + Shift + R`）で録音を開始/停止します。
- トレイ常駐するので、ウィンドウを閉じても裏で待機します。設定で **PCログイン時の自動起動** も可能。

### 録音モードと録音ソース
- **録音モード**: 「トグル（1回で開始/停止）」または「**押している間だけ録音**（モーメンタリ）」。
- **録音ソース**: マイクのほか、**システム音（PCが再生している相手の声・音）** や、**マイク＋システム音の同時録音**を選べます（Windows）。

### 物理ボタン連携
マウスのサイドボタン・フットスイッチ・Stream Deck などに、QuickScribe のホットキーを割り当てるだけで連携できます。詳細はリポジトリの `docs/guide/physical-triggers.md` を参照してください。

## 3. 文字起こし → 用語チェック → 整形

1. 録音を止めると **文字起こし** が走ります（ローカル whisper、またはクラウドSTT）。
2. 必要に応じて **「✓ 用語チェック」**: 誤変換が疑われる語（例「レーメン」→「LLM」）をAIが検出し、置換を提案します。置換する/しないを選んで適用できます。
3. **「✨ 整形する」**: ニュアンスを残しつつ、**構造化 / 逐語 / 要約 / ブレスト**、または**自作のカスタムパターン**で思考を整理します。

## 4. ジャーナルで振り返る

- エントリは **出力先フォルダ**（既定: ドキュメント/QuickScribe）に保存されます。生の文字起こしは `transcript-…`、整形済みは `refined-….md` と名前で区別。
- ヘッダの **ジャーナル** から、過去エントリの**一覧・全文検索・タグ絞り込み**ができます。
- **「✨ 横断発見」**: 絞り込んだ複数エントリをAIが読み解き、繰り返すテーマ・感情の傾向・未解決の問い・次の一歩を抽出します。
- 整形結果の操作列の **「出力先を開く」**（または設定）から、保存先フォルダをそのまま開けます。

## 5. 設定

- **文字起こしエンジン**: ローカル whisper（`base` / `kotoba-whisper`〔日本語特化〕など選択可）/ クラウド（Groq・OpenAI・Deepgram・Azure）。
- **整形プロバイダ**: Gemini / Anthropic / OpenAI / ローカル Ollama / AWS Bedrock / Claude Platform on AWS。**鍵はOSのセキュアストレージに保管**。
- 出力形式（txt / Markdown）、ホットキー、自動起動 など。

> クラウド連携を使う場合のみ、対象データが各プロバイダへ送信されます。プライバシー重視なら「ローカル」をお使いください（[プライバシーポリシー](/privacy)）。
